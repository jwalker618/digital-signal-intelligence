## Overview

Final phase. Single comprehensive rewrite of `seed_dsi_bench.py` that exercises
every v5 production code path. The seed script is both a database population tool
and a **production validation harness** -- every computed value flows through the
real engine, never fabricated. This is the last phase executed; no other phase
modifies the seed script.

## Current Seed Script

`seed_dsi_bench.py` seeds 160+ companies across 10 coverages and 76 configs using
production components: `ModelScorer`, `ModelPricer`, `QueryEvaluator`, `PremiumAssembler`,
`FXConverter`, `ROLValidator`, `ROLRecommender`. Each company creates: Submission,
ModelVersionRecord, ModelVersionSignal, Quote, Referral, SignalCache, SignalAuditRecord,
AuditLog, CommercialTermsRecord, RiskTermsRecord, ConfigSnapshot.

## Principle

**Every computed field flows through production code.** Seed data for new tables
is generated by running the real engine, not by inserting fabricated rows. The
seed script doubles as an end-to-end integration test.

### Phase WE-4: Causal Graph Pricing

**What to seed**: CAF computation for all seeded assessments.

```python
# Production path: CausalPricingEngine.compute_caf()
from world_engine.causal_pricing.engine import CausalPricingEngine

causal_engine = CausalPricingEngine(registry)

for mv_record in all_model_version_records:
    caf_result = causal_engine.compute_caf(
        entity_id=mv_record.entity_id,
        signal_scores=_extract_signal_scores(mv_record),
        derivative_values=_extract_derivatives(mv_record),
        current_tier=mv_record.final_tier,
        tier_rates=_get_tier_rates(mv_record),
    )
    registry.store_caf(caf_result)

    # Update the model version record with CAF
    mv_record.caf_value = caf_result.caf_value
    mv_record.caf_confidence = caf_result.confidence
```

Entities with active precursors (from WE-3 seeded relationships) will show CAF != 1.0. Entities without precursors show CAF = 1.0. Both states represented.

## Seed Script Execution Order

The seed script must run subsystems in dependency order:

```
1. Auth foundation (tenant, users, roles)                    [A-1]
2. Core submissions pipeline (existing -- 160+ companies)     [existing]
3. World Engine foundation (maturity evaluation)              [WE-1]
4. Consistency scoring (all assessments)                      [WE-2]
5. Re-assessments (temporal data for subset of entities)      [WE-3 prereq]
6. Discovery pipeline (scanner, lifecycle)                    [WE-3]
7. CAF computation (all assessments)                          [WE-4]
8. Portfolio graphs + concentration detection                 [WE-5]
9. Scenario simulation (demo)                                 [WE-5]
10. Loss event ingestion + signal-loss linking                [C-1]
11. Recalibration engine (generate proposal)                  [C-2]
12. Audit events (for key actions across all above)           [A-2]
13. Config version history                                    [B-2]
14. Pipeline metric snapshots                                 [B-1]
15. Maturity re-evaluation (final state)                      [WE-1]
```

## Constraints

1. Every computed value MUST flow through production code -- no fabricated computed fields
2. Synthetic data (signal scores, loss amounts) is OK; computed outputs (premiums, CAF, consistency, proposals) must come from real engines
3. Seed script must complete in < 120 seconds
4. Seed script must be idempotent (drop and recreate on re-run)
5. Failures in optional subsystems (WE, recalibration) should log warnings, not abort the entire seed
