DOCUMENT_NAME: Global Rules & System Architecture
DOCUMENT_ID: global_rules_system_architecture
DOCUMENT_TYPE: master_specification
APPLIES_TO: all topics, all layouts, all components, all interactions

====================================================================
GLOBAL LAYOUT RULES
====================================================================

PAGE_STRUCTURE:
  - full_page_background_canvas (dynamic network)
  - fixed_header_bar
  - content_panel (centered, responsive)
  - footer (minimal, optional)

SAFE_AREAS:
  - 24px padding on all sides
  - 48px minimum spacing between header and content_panel
  - 32px minimum spacing between content blocks

CONTENT_PANEL:
  WIDTH:
    desktop: 760–980px
    tablet: 600–760px
    mobile: 100% width (background disabled)
  BACKGROUND: #FFFFFF
  BORDER_RADIUS: 12px
  SHADOW: subtle, low elevation
  SCROLL_BEHAVIOUR: internal scroll disabled (topics use section navigation)

====================================================================
GLOBAL NAVIGATION RULES
====================================================================

HEADER_BAR:
  HEIGHT: 64px
  POSITION: fixed_top
  BACKGROUND: #FFFFFF (95% opacity)
  BORDER_BOTTOM: 1px solid #E5E5E5
  Z_INDEX: 1000

HEADER_CONTENT:
  LEFT_SIDE:
    - HOME_BUTTON (icon + text)
    - POSITION: far_left
    - ICON: home_symbol.svg
    - TEXT: "Home"
    - INTERACTION: navigate_to_home
    - NOTE: This replaces the previous multi-topic header navigation.

  RIGHT_SIDE:
    - GENERATE_LOGO (see asset rules)
    - POSITION: top_right
    - SIZE: 32–40px height
    - BEHAVIOUR: static, non-interactive

REMOVED_ELEMENTS:
  - No topic list in the header (Automation Myth, Core Thesis, etc.)
  - No horizontal navigation between topics at the global level

TOPIC_NAVIGATION:
  - occurs *inside* each topic
  - next_section / previous_section buttons
  - chevron navigation for section jumps
  - no global topic switching from header

====================================================================
GLOBAL ASSET RULES
====================================================================

LOGO_REQUIREMENTS:
  - The Generate logo must be provided as a .png file.
  - Expected format: PNG, transparent background, 2× resolution recommended.
  - Developer workflow:
      1. User provides correct .png asset.
      2. Asset stored in /assets/logo/generate_logo.png
      3. Header references this asset globally.

LOGO_PLACEMENT:
  - top_right of header bar
  - fixed position
  - padding-right: 24px
  - padding-top: 12px
  - height: 32–40px
  - width: auto

ASSET REPLACEMENT WORKFLOW:
  - system must allow user to upload a replacement logo
  - replacement overwrites existing asset
  - cache invalidation required on next page load

====================================================================
GLOBAL VISUAL SYSTEM RULES
====================================================================

BACKGROUND_CANVAS:
  TYPE: dynamic_network_canvas
  POSITION: full_page_behind_content
  OPACITY: 0.15–0.25
  MOTION: subtle, non-distracting
  MOBILE: disabled (performance + clarity)

EVOLUTION_MODEL:
  - background evolves per section according to VISUAL_STATE
  - transitions animate_on_section_change
  - canvas persists across sections within a topic
  - canvas resets when navigating to a new topic

VISUAL_STATE PARAMETERS:
  - NODE_COUNT: integer range
  - EDGE_DENSITY: very_low ? maximum
  - PALETTE: primary + secondary colours
  - MOTION_SPEED: very_slow ? fast
  - BEHAVIOUR: descriptive rules for node/edge behaviour

====================================================================
GLOBAL COMPONENT LIBRARY
====================================================================

COMPONENTS:
  - static_card
  - expandable_card
  - side_by_side_card
  - two_card_row
  - three_card_row
  - four_card_grid
  - five_card_grid
  - seven_card_grid
  - ten_card_grid
  - section_header
  - section_supporting_text
  - navigation_chevrons
  - next_section_button
  - previous_section_button

STATIC_CARD:
  - fixed height (auto)
  - title + body
  - no interaction

EXPANDABLE_CARD:
  - collapsed_state: title + truncated body
  - expanded_state: full body + validation fields (if present)
  - animation: smooth vertical expansion
  - collapse_on_second_click: true

SECTION_NAVIGATION:
  - next_section button at bottom of each section
  - previous_section button at top-left of each section
  - chevron navigation for multi-section topics

====================================================================
GLOBAL INTERACTION RULES
====================================================================

INTERACTION_MODEL:
  - deterministic
  - no hidden states
  - no randomisation
  - all interactions must be reversible

ACCESSIBILITY:
  - keyboard navigation required
  - ARIA labels for all interactive elements
  - minimum contrast ratio: 4.5:1

MOBILE RULES:
  - background canvas disabled
  - header remains fixed
  - content_panel becomes full-width
  - cards stack vertically

====================================================================
TOPIC-TO-TOPIC TRANSITION RULES
====================================================================

TRANSITION_MODEL:
  - fade_out current topic
  - fade_in next topic
  - background canvas resets to topic’s SECTION 1 VISUAL_STATE
  - header remains static (Home left, logo right)

ENTRY_BEHAVIOUR:
  - scroll position resets to top
  - section index resets to 1

EXIT_BEHAVIOUR:
  - no scroll persistence
  - no background persistence across topics

====================================================================
NAMING CONVENTIONS
====================================================================

TOPIC_IDS:
  - lowercase
  - underscores only
  - example: three_layer_assessment

SECTION_IDS:
  - lowercase
  - underscores only
  - example: risk_layer

ASSET_NAMES:
  - lowercase
  - hyphens allowed
  - example: generate-logo.png

====================================================================
IMPLEMENTATION CONSTRAINTS
====================================================================

PERFORMANCE:
  - background canvas must run at <10% CPU on modern laptops
  - animations capped at 30fps
  - no heavy DOM operations during transitions

SECURITY:
  - no external script injection
  - all assets loaded from local bundle
  - no remote code execution

AGENTIC READINESS:
  - all content must be LLM-readable
  - all structures must be machine-parseable
  - no ambiguous formatting
  - deterministic section ordering
