DSI EXPOSURE SHADOW LAYER – FULL SPECIFICATION (TEXT VERSION)
=============================================================

This document defines the complete technical specification for the DSI Exposure Shadow Layer. It is designed for future implementation by an LLM or engineering team. It is coverage‑agnostic, actuarially valid, and fully aligned with DSI principles: externally observable signals, machine‑readable inputs, network authority over self‑reporting, and full auditability.

----------------------------------------------------------------------
SECTION 1 — PURPOSE
----------------------------------------------------------------------

The Exposure Shadow Layer provides a fully automated, DSI‑native method for estimating exposure without requiring traditional bordereaux, SOVs, geolocation, or manual asset schedules. It infers exposure bands using digital footprint signals, network authority, geo‑intensity, distribution proxies, and cohort priors.

The output is:
- Exposure Score (0–100)
- Exposure Band (micro, small, medium, large, very_large)
- Exposure Confidence (0–1)
- Exposure Modifiers (optional)
- Exposure‑driven referral triggers
- Cohort‑implied TIV range (optional)

This layer does NOT replace risk scoring. It runs in parallel and influences pricing and decisioning through explicit, auditable mechanisms.

----------------------------------------------------------------------
SECTION 2 — EXPOSURE SIGNALS AND DATA CONTRACTS
----------------------------------------------------------------------

Each exposure signal must be externally observable, machine‑readable, and reproducible. No client‑provided data is required.

All extractors must return the following structure:

signal_id: string
value: number | string | boolean
normalized_value: number (0–100) or null
confidence: number (0–1)
source_urls: list of strings
extracted_at: ISO8601 timestamp
notes: optional string

Canonical exposure signals:

1. domain_count
2. subdomain_complexity
3. tech_stack_count
4. global_presence_score
5. web_reach_proxy
6. marquee_partner_count
7. regulatory_citation_count
8. systemic_link_score (PageRank‑style)
9. location_count
10. high_intensity_location_pct
11. broker_tier
12. programme_complexity_flag

Extractors must be idempotent, cacheable, and return confidence scores.

SECTION 5 — NORMALIZATION AND SCORING LOGIC
----------------------------------------------------------------------

Normalization functions:

1. log_scale:
   score = min(100, log(value, base) * scale)

2. linear:
   score = 0 if value <= min
   score = 100 if value >= max
   else proportional

3. capped:
   score = min(value, cap) / cap * 100

4. percentile:
   score = percentile_rank(value, cohort_distribution)

5. categorical_map:
   score = mapping[value]

Group scoring:

group_score = weighted average of normalized feature scores
group_confidence = weighted average of feature confidences

Exposure composite score:

exposure_score = weighted average of group_scores

Exposure confidence:

exposure_confidence = weighted average of group_confidences

----------------------------------------------------------------------
SECTION 6 — EXPOSURE BAND MAPPING
----------------------------------------------------------------------

Two mapping methods:

1. fixed_threshold:
   Compare exposure_score to min_score/max_score ranges.

2. cohort_quantile:
   Use cohort priors to map score percentiles to exposure bands.

Cohort priors are derived from historical portfolio data:
- Sector
- Digital footprint band
- Region
- Network authority band

Each cohort stores:
- empirical TIV distribution
- empirical loss severity distribution
- mapping from exposure_score percentile to implied TIV band

----------------------------------------------------------------------
SECTION 7 — PRICING INTEGRATION PATTERNS
----------------------------------------------------------------------

Three supported patterns:

PATTERN A — Parallel View (recommended)
- Exposure does not affect risk score.
- Exposure band influences pricing via modifiers or ILF selection.

PATTERN B — Embedded Weight
- Exposure groups included in composite score.
- Only used when actuarially validated.

PATTERN C — Two‑Dimensional Pricing Grid
- Pricing determined by (risk tier, exposure band) matrix.

SECTION 8 — AUTO APPLY RULES
----------------------------------------------------------------------

Rules defined in YAML:

Examples:

1. High exposure + low confidence → referral
2. Large exposure band → apply modifier
3. Very large exposure band → minimum tier override (optional)

Rules must be:
- deterministic
- auditable
- logged in ModelVersion
